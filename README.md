# Spec-to-Sell

Spec-to-Sell is an evidence-first product-visual studio for print-on-demand sellers. It turns supplier specifications and vendor photos into a focused 2–6 image listing story—with a proof trail for every claim and optional GPT Image 2 art direction for each product visual.

## The product promise

PoD sellers inherit scattered supplier text and underwhelming vendor photos, then have to make a credible listing gallery quickly. Spec-to-Sell makes the workflow useful without pretending a generated image is an original product photograph.

- Every exportable product claim has an exact supplier-text citation or a specific vendor-photo citation.
- Unsupported claims are held from the storyboard until the seller supplies exact evidence.
- Sellers can choose the original vendor photo or generate an **AI-composed visual** using GPT Image 2 with the supplied vendor photos as high-fidelity product references.
- AI-composed slides are visibly labelled. The model is explicitly told not to render text, claims, labels, logos, or watermarks; Spec-to-Sell layers the brand typography and evidence-backed text deterministically afterward.
- The adaptive storyboard selects 2–6 slides from the available evidence instead of forcing every product into a fixed set.

This keeps the most important distinction clear: the visual may be art-directed by AI, but the product claims are still source-cited and the exact exported text is never generated by the image model.

## Guided flow

1. Set a compact brand direction: name, voice, accent colour, and optional logo.
2. Add 1–8 vendor photos and the factual supplier specifications.
3. Analyze the proof trail with GPT-5.6 Terra, then review supported and held claims.
4. Plan the listing story.
5. Generate an active slide’s product composition with GPT Image 2, or retain the original vendor source.
6. Export one 2000 × 2000 PNG or the complete ZIP.

The packaged tote demo works without an API key for the source-only and evidence-review experience. With an API key, it can also be rasterized and sent to GPT Image 2 as a demo reference image.

## Run locally

    npm install
    copy .env.example .env.local
    npm run dev

Open http://localhost:3000. Add a server-side key to use live analysis and image generation:

    OPENAI_API_KEY=your_key_here

Never commit the key or use a `NEXT_PUBLIC_` variable. Both OpenAI calls are server-only.

## OpenAI implementation

- `POST /api/analyze` uses GPT-5.6 Terra through the Responses API to produce source-cited facts and a deterministic storyboard.
- `POST /api/generate-visual` uses [`gpt-image-2`](https://developers.openai.com/api/docs/models/gpt-image-2) through the Image Edits API.
- The visual route validates every input photo, the brand brief, the active slide, and up to three supported facts. Supplier-text citations must be exact substrings of the supplied specifications; vendor-image citations must point to an uploaded photo.
- GPT Image 2 receives the vendor photos as high-fidelity references. Its prompt prohibits text, logos, unreferenced product changes, unsupported product features, and people unless a person is in a supplied vendor image.
- The response is a compressed WebP preview; React Konva composes it with exact text and exports a 2000 × 2000 PNG. AI composition never changes the claim layer.

## Vercel deployment

1. Import this GitHub repository into Vercel using the Next.js preset.
2. Add `OPENAI_API_KEY` in **Preview** and **Production** environment variables. Do not expose it to the client.
3. Redeploy after saving the key.
4. In **Vercel Firewall → Custom Rules**, create these fixed-window rate-limit rules (key: IP address):
   - `POST` + request path `/api/analyze`: **5 requests per 10 minutes**.
   - `POST` + request path `/api/generate-visual`: **3 requests per 10 minutes**. This endpoint is intentionally tighter because it invokes image generation.
5. Verify the no-key demo, a live analysis, one AI visual, and PNG ZIP export before sharing the link.

## Verification

    npm test
    npm run build

The test suite covers storyboard bounds, exclusion of unsupported claims, evidence validation, malformed or oversized analysis input, and visual-generation request grounding.

## Codex and OpenAI

Codex accelerated the evidence model, server validation, guided studio, GPT Image 2 visual-generation integration, deterministic export layer, tests, and deployment documentation. GPT-5.6 Terra is used for multimodal supplier-fact extraction and storyboard planning. GPT Image 2 art-directs vendor-referenced product compositions; it does not determine which claims may be exported.
